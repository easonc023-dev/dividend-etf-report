什么是 Skill？

给你专属的 AI 使用说明书——让 Claude Code 在工作时自动翻看，干活更精准、更专业。

一个 Skill 就是一份“专属说明书”，安装后 AI 会在恰当的时机自动翻阅它，照着里面的规范、流程、工具来做事。

就像你给新同事发了一份《品牌 VI 手册》——他做 PPT 时翻一下，就知道颜色用哪个、字体用哪个。Skill 对 AI 做的是一模一样的事。

下面拿 brand-guidelines（品牌配色规范）举例子，看看一个 Skill 从哪 7 个维度定义自己：

1
元数据
叫什么？什么时候触发？
例： "用户提到品牌配色时自动激活"
2
边界
管什么、不管什么
例：管颜色字体，不管动画
3
流程
分几步干完活
例：选色 → 选字 → 应用 → 检查
4
工具
靠什么干活
例：色卡 (#141413 等)、字体栈
5
容错
条件不够时怎么办
例：字体没装？降级到 Arial 这个很关键
6
评测
怎么知道干好了没
例：标题是 Poppins 吗？颜色是 #141413 吗？
7
精准
会误触发吗？漏触发吗？
例：说"配色"触发，说"调色"不触发

加载别人的技能

3 种方式，从简单到高级

简单

方式一：GitHub 仓库克隆

从 GitHub 克隆技能仓库，复制到全局技能目录。适合安装 Anthropic 官方或社区技能。

git clone https://github.com/anthropics/skills /tmp/skills
cp -r /tmp/skills/skills/技能名 ~/.claude/skills/技能名
中等

方式二：手动创建

新建目录 + SKILL.md 文件，适合少量简单技能。

mkdir -p ~/.claude/skills/my-skill
# 然后创建 SKILL.md，写上前置元数据和内容
# 参考下方「创建模板」
高级

方式三：skill-creator 自动生成

使用 skill-creator 元技能，带 eval 验证和迭代优化，适合生产级技能。

# 直接对我说：
"用 skill-creator 帮我创建一个技能，用来……"

自己创建新技能

7 维清单 + 最小模板，拿来即用

  • 想好名字和触发条件（description 字段最重要）
  • 明确技能边界：做什么、不做什么、需要什么环境
  • 写出分步执行流程（每一步输入→输出→判断）
  • 列出依赖工具和脚本
  • 想好出错时的 fallback 方案
  • 写 3-5 个可验证的测试断言
  • 补充 SKIP 条件，避免误触发

👆 点击每个条目标记完成

最小可运行模板

--- name: my-first-skill description: "一句话说清功能 + 触发条件。例：当用户说X时触发，当用户只是Y时跳过。" --- # 技能名称 ## 概述 这个技能用来解决什么问题。 ## 执行步骤 ### Step 1：检查环境 确认 XXX 可用，否则提示用户安装。 ### Step 2：执行核心任务 1. 读取输入 2. 处理数据 3. 输出结果 ### Step 3：验证 检查输出是否符合预期格式。 ## 异常处理 - 如果 A 失败 → 走 B 方案 - 如果 C 不存在 → 提示用户
