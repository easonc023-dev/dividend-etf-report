## 三个问题，一个根因

**第一，数据口径是乱的。**

ETF有场内行情价（不含分红），场外基金有单位净值，还有累计净值（含分红）。你把两支产品放一起对比收益率，根本不知道比的是不是同一件事。这就像拿一个人的税前工资和另一个人的税后工资比谁赚得多——没有意义。

**第二，买卖时机全凭拍脑袋。**

网上说"RSI低于30是超卖，可以抄底"。但这个30是怎么来的？对所有产品都一样吗？对同一支产品，历史上RSI到30以下之后，到底涨了还是继续跌？——没人告诉你。包括那些推荐你买基金的App。

*大多数投资建议的问题在于：它们告诉你要做什么，但从不说这个建议是怎么来的。*

**第三，信息分散得像拼图。**

估值在东方财富，技术指标在券商App，分红记录要翻公告，持仓行业要手动整理。每次想做一个稍微认真的决策，都要在五六个平台之间切来切去。切换成本高到——算了，不分析了，随便买吧。

这三个问题的根因其实是一个：

**缺少一个"单一事实来源"——把所有数据汇到一起，用统一口径计算，输出可对比、可追溯的信号。**

就像做饭，食材分散在各个超市不可怕；可怕的是连砝码都不统一，你根本不知道一斤肉和一斤菜谁更重。

## 四个决策，从0到1

于是开始动手。从最初一支ETF的深度分析，到后来11支产品的量化决策中枢，中间做了四个关键决定。

### 一、别写一次性脚本，写框架

最早只为南方红利低波50（515450）做了一个页面。跑通之后没有停，而是立刻把逻辑泛化：用Tab式结构支持多产品切换。

现在覆盖11支产品，按"能源资源红利"和"常规红利"分两类。新增一支产品只需要加一行配置——不需要重写任何逻辑。

*一次性脚本解决今天的问题，框架解决未来的问题。很多人不愿意多花那30%的时间做泛化，结果后面花300%的时间在维护。*

### 二、别拍脑袋，跑回测

RSI参数到底怎么选？6天还是14天？观察点设在20还是30？

不靠经验，靠回测。

设计了一个矩阵：3个日线周期 × 5个观察点 + 2个周线周期 × 5个观察点 = **25组参数组合**，再 × 7个持有期（5日到550日） × 11支产品。用历史数据逐个跑，出年化收益、单次收益、胜率、信号频率。

回测跑完，一个规律清晰浮现：**收益和频率是跷跷板。**

- 参数调灵敏（6日RSI / 观察点25）：年化+17.4%，胜率79%，年均5.9次信号——但单次只赚2.9%
- 参数调迟钝（7周RSI / 观察点40）：年化+6.6%，胜率70%，年均1.9次——但单次能赚3.4%

没有绝对的好坏。高频选手赚的是次数，低频选手赚的是确定性。

### 三、别混为一谈，分开评价

回测结果逼出了一个"双路径框架"：

**路径一：高频捕捉小波段。** 6日RSI / 观察点25。看"年化预期收益"——信号密、频率高，适合资金周转快的人。

**路径二：低频等待大机会。** 7周RSI / 观察点40。看"单次信号质量"——信号稀但确定性高，适合单笔资金大、追求稳妥的人。

这是最容易犯错的地方：*你不能用同一个标准去评价两个完全不同的策略。*

高频策略嫌它单次收益低？那是你没理解它的逻辑——它赚的是复利叠加，不是单次暴击。低频策略嫌它信号太少？那是你忘了它的优势——每次出手时，你有更高的信心下更大的注。

顺便说一句，14周RSI被直接丢弃——回测没进前15，信号太稀疏，对红利策略没有实操价值。*数据不会照顾你的感受，该扔就扔。*

### 四、别混用口径，统一尺子

这个决定最技术性，但也最关键。

所有11支产品的数据源，统一为**累计净值（含分红再投资）**。不管你是ETF还是场外基金，全部用同一个指标——总回报——来衡量。

这样回测脚本和前端报告的计算口径完全一致。杜绝了"回测算出来好看，实盘对不上"的隐患。

*口径统一这件事，不性感，不重要到让人忽略，但忽略它会让你所有数据失去意义。*

## 零运维：一个人也能跑的管线

整个系统就四步，三个免费工具：

```
数据采集（akshare 天天基金 + baostock 估值分红）
    ↓
Python 计算引擎（指标 + 评分 + 回测，全在单文件里）
    ↓
静态 HTML 报告（内联CSS + 原生JS，不依赖任何框架）
    ↓
GitHub Pages 自动部署（Windows 定时任务每日触发）
```

选型原则：**不需要服务器、不需要数据库、不需要付费 API。**

每天下午4点07分，系统自动采集数据 → 计算 → 生成报告 → Git提交 → GitHub Pages上线。全程无人值守。我甚至不需要打开电脑。

## 系统到底能干什么

每支产品展示10个维度的数据：累计净值、PE/PB估值（加权平均）、股息率、RSI 6日/7周、MA250/MA550偏离、年化波动率、最大回撤、多阶段收益、行业分布、持仓明细。

几个关键设计：

**复合买入评分（0-100分）。** RSI6 + RSI7W + MA250 + MA550 四个信号加权合成一个分数。≥45标记买入，30-44观察，<30安全。不用自己对着四个指标纠结——让权重替你判断。

**RSI历史百分位。** 不只告诉你RSI=35，还告诉你这个35在历史中处于什么位置。第8百分位意味着比92%的时间都便宜。历史坐标系比绝对数值有用十倍。

**信号规则（Rule A）。** RSI跌破观察点触发一次，必须回升超过观察点+5才重新"上膛"。这样在持续超卖区间不会反复触发假信号。

**同类对比+策略卡片。** 8支红利ETF横向排名。每支产品独立展示跟踪什么指数、用什么因子选股、怎么加权、多久调一次仓。
